# Two CVEs are suppressed in audit-python.sh for this service. They are present
# as transitive deps of marker-pdf, which pins older versions of both packages.
# Upgrading them directly causes a dependency conflict (marker-pdf 1.10.2 requires
# Pillow<11.0.0 and weasyprint<64.0). Remove the suppressions once marker-pdf
# depends on pillow>=12.1.1 and weasyprint>=68.0.
#
# CVE-2026-25990 (pillow, CVSS 8.9 High):
#   Out-of-bounds write when loading PSD (Photoshop) images. Exploitable only if
#   a PDF containing a crafted PSD image is processed. PDFs do not normally embed
#   PSD files; risk is low for typical document sources (web pages, bank statements).
#
# CVE-2025-68616 (weasyprint, CVSS 7.5 High):
#   SSRF bypass via HTTP redirects — a crafted document can cause WeasyPrint to
#   fetch internal URLs (e.g. localhost services). Risk is low in a single-user,
#   self-controlled deployment with no cloud metadata endpoints, but would be
#   significant if this service were ever exposed to untrusted document sources.
